背景：两条核心问题线

大模型 RL 训练中，随着规模扩大和架构演进（尤其是 MoE），出现了两大核心挑战：

  1. Off-policy 裁剪问题：当 rollout 策略与训练策略产生偏差时，importance sampling ratio 的处理方式直接影响训练稳定性
  2. MoE 训推不一致：MoE 模型中，推理引擎和训练引擎的 routing 决策差异导致概率分布不匹配，积累后可导致训练崩溃

演进线 1：Off-policy 裁剪问题 PPO → DAPO → CISPO

PPOProximal Policy Optimization

核心机制

对 importance sampling ratio r(θ) = πθ / πold 做对称裁剪 clip(r, 1-ε, 1+ε)，将策略更新限制在信任域内。

解决的问题

防止策略更新过大导致训练崩溃。

在 LLM RL 中的不足

  • 对称裁剪在 long-CoT 场景下导致 entropy collapse（熵坍缩）——模型过早变得确定性，丧失探索能力
  • 关键的“分叉 token”（触发反思/修正的 rare token）在 base policy 下概率低，importance ratio 大，容易被裁剪掉，梯度贡献归零
  • 多 minibatch 训练中，稀有但重要的 token 在第一次 on-policy 更新后就被 clip 掉

DAPODecoupled clip and Dynamic sAmpling Policy Optimization

ByteDance Seed & 清华 AIR — arXiv:2503.14476

核心创新

解耦上下裁剪边界 + 动态采样

四个关键技术

Clip-Higher
非对称裁剪，上界 0.28 / 下界 0.2

解决 entropy collapse：允许增大概率的更新有更大空间，保持探索多样性

Dynamic Sampling
过滤 acc=0/1 的 prompt

提升训练效率：只保留有有效梯度的样本

Token-Level PG Loss
token 粒度策略梯度

在 long-CoT 场景增强训练稳定性，让长度增长更健康

Overlong Reward Shaping
渐进惩罚而非硬截断

减少奖励噪声：对过长 response 施加渐进惩罚而非硬截断

主要贡献

识别并解决了 PPO 对称裁剪在 long-CoT 场景的 entropy collapse 问题。

CISPOClipped Importance Sampling Policy Optimization

MiniMax-M1 项目 — arXiv:2506.13585

核心创新

裁剪 IS 权重本身，而非丢弃 token

与 PPO/GRPO 的关键区别

PPO/GRPO

ratio 超出阈值时，整个 token 的梯度贡献归零（hard token dropping）

vs
CISPO

对 IS 权重做 cap，token 仍然贡献梯度，只是权重被上限截断（soft weight saturation）

解决的核心问题

保留稀有但重要的 token 的学习信号。在 long reasoning trace 中，那些触发反思/修正的“分叉 token”概率低、ratio 大，PPO 会丢弃它们，CISPO 确保它们始终参与梯度计算。

已知问题

  • 后期可能出现 model collapse（性能急降 + 熵快速下降）
  • 不考虑训推不一致问题
  • length normalization 可能使 first-order approximation 失效

演进线 1 对比总结

方法 裁剪策略 对 rare token 的处理 熵控制 适用场景
PPO 对称 clip ratio 丢弃（梯度归零） 无特殊处理，易 collapse 短序列、dense model
DAPO 非对称 clip ratio（上宽下窄） 仍可能丢弃，但上界更宽松 Clip-Higher 缓解 collapse Long-CoT reasoning
CISPO clip IS weight（非 ratio） 保留所有 token 贡献 无显式控制，后期可能 collapse 大规模 off-policy 训练

演进线 2：MoE 训推不一致 IcePop → R3 → MIS-PO → GLM-5 Slime

问题本质

MoE 模型中，router 动态选择 top-k expert 处理每个 token。现代 RL 框架通常使用不同引擎做推理和训练（如 SGLang 做 rollout，Megatron 做 training），导致：

~10%
router 层选择了不同的 expert
94%
token 至少在一个层选了不同 expert
~6
每个 token 平均有约 6 个 router 选择不同

这种不一致在 long-CoT 中逐步累积和放大，导致梯度不稳定甚至训练崩溃。

IcePop

Ring-1T（万亿参数模型训练） — arXiv:2510.18855

核心机制

Token 级掩码 + 选择性梯度裁剪

两个关键技术

Double-sided Calibration

在上下界范围内校准 token 级梯度，保持训推概率的对齐

Masking

直接排除概率偏差过大的 token，不参与梯度计算

定位

独立于 sequence-level 优化，可更广泛地应用。

效果

Ring-1T 用 IcePop 在 AIME-2025 达 93.4，梯度范数和 KL 散度全程有界。

局限

丢弃（mask 掉）偏差大的 token，信息损失；本质是“回避问题”而非“解决问题”。

R3Rollout Routing Replay

小米 & 北大 — arXiv:2510.11370

核心创新

从源头解决问题——记录推理引擎的 routing 决策，在训练时重放

机制

在 rollout 阶段捕获 routing distribution，训练时强制训练引擎使用相同的 routing mask，从而消除训推之间的 routing 差异。

vs 其他方法

GSPO / TIS

在 routing 决策之后做下游修正（importance sampling 校正）

vs
R3

直接对齐 routing 决策本身（上游根治）

效果

显著降低训推 KL 散度，防止 RL collapse，不影响训练速度。

局限

需要额外存储和传输 routing 信息；routing replay 使训练时 router 无法自适应更新（冻结了 routing 决策）。

MIS-POMetropolis Independence Sampling-Filtered Policy Optimization

StepFun (Step 3.5 Flash) — arXiv:2602.10604

核心创新

二值过滤 + routing 置信度监控

关键机制

Binary Filtering
二值过滤替代 IS

用 likelihood-ratio 做二值接受/拒绝，不做连续 IS 权重校正，降低梯度方差

Routing Confidence Monitoring
routing 置信度监控

监控 MoE routing 稳定性，确保训练过程中 routing 行为一致

Truncation-Aware Value Bootstrapping
截断感知价值引导

对撞到 context limit 的长序列不做惩罚，保证公平的 credit assignment

定位

同时解决 off-policy drift 和 MoE routing 不稳定，是一个统一框架。

效果

Step 3.5 Flash (11B active / 196B total) 在 AIME 2025、HMMT 2025 等达到 frontier 水平。

局限

二值过滤丢弃了大量样本（被 reject 的完全不用），数据利用效率可能不如 CISPO 等连续权重方法。

GLM-5 Slime 框架

智谱 AI (Z.ai) — arXiv:2602.15763

核心创新

全异步 RL 架构 + IS 校正 + 确定性 top-k routing

关键机制

异步解耦
rollout 与 training 完全独立

消除同步瓶颈，GPU 利用率接近 100%，吞吐量达同步方法的 3x

IS Correction
Importance Sampling 校正

校正 behavior policy 和 training policy 之间的分布偏移

Deterministic Top-k Routing
确定性路由

用确定性 top-k 替代随机 routing，从根本上消除 routing 随机性导致的训推不一致

APRIL
Active Partial Rollouts

解决 rollout 长尾延迟问题

定位

系统级解决方案，同时处理 off-policy 问题（IS 校正）和 MoE 不一致问题（确定性 routing）。

效果

GLM-5 (744B / 40B active)，HLE 达 50.4%，幻觉率从 90% 压缩至 34%。

局限

确定性 routing 可能牺牲 MoE 的负载均衡灵活性；全异步架构的工程复杂度高。

跨线对比：各方法解决的问题分类

方法 解决 off-policy 裁剪 解决 MoE 训推不一致 具体子问题

个人评价：优劣对比

演进线 1（Off-policy 裁剪）

DAPO

DAPO 的价值在于清晰地识别了 entropy collapse 问题，Clip-Higher 是一个简洁优雅的方案。但它本质上还是在 “ratio clipping” 的框架内做改良，没有跳出 PPO 的范式。

CISPO

CISPO 的思路更彻底——不丢弃任何 token，只 cap 权重。这在理论上更合理，因为 rare-but-important token 正是 long reasoning 中最关键的信号。但它的 entropy 控制是软肋，后期 model collapse 的问题限制了其在超长训练中的适用性。

判断

CISPO 的方向更正确（保留所有 token 信号），但需要配合显式的 entropy 正则化。DAPO 在工程上更成熟和稳定，适合当前阶段的落地。

演进线 2（MoE 训推不一致）

IcePop

IcePop 是“治标”——mask 掉偏差大的 token，信息损失但训练稳定。适合作为快速修复方案。

R3

R3 是“治本”——直接对齐 routing 决策。思路最清晰，效果好，但冻结了 routing 的自适应能力，且需要额外的存储/传输开销。

MIS-PO（Step 3.5 Flash）

MIS-PO 尝试统一解决两个问题，但二值过滤太过激进，数据利用效率不高。routing confidence monitoring 是好想法但细节公开不多。

GLM-5 Slime

GLM-5 的 deterministic top-k routing 是最激进也最彻底的方案——直接消除随机性。这牺牲了 MoE routing 的动态灵活性，但换来了训推的严格一致。异步架构的 3x 吞吐提升在工程上非常有价值。

判断

  • 短期最佳实践：R3（简单、直接、有效，且有严格的实验验证）
  • 长期方向：GLM-5 的 deterministic routing + 异步架构思路更有前景，但工程门槛高
  • MIS-PO 是一个有意义的统一尝试，但公开细节不足，难以评估其真实优势

整体趋势

  1. 从 clip ratio → clip IS weight → binary filter → deterministic routing：方法越来越激进，从“限制更新幅度”到“直接消除不一致源头”
  2. 从单一问题 → 统一框架：早期方法（DAPO、IcePop、R3）各解决一个问题，后期方法（MIS-PO、Slime）试图统一解决
  3. 系统工程越来越重要：GLM-5 的 Slime 框架表明，RL 训练的瓶颈已不仅是算法，更是基础设施和系统设计
